读fody.PropertyChanged源码（4a）：EventInvokerMethod对象

Feb 11, 2018

假设我们有如下类Customer，它包含一些属性。

public class Customer
{
  public string Name { get; set; }
  public string Phone { get; set; }
  ....
}

当Name属性被修改时，我们需要调用NotifyPropertyChanged()方法来通知 该属性有了新的值。如果我们希望fody.PropertyChanged来做这件事情，它必须 先找到这个NotifyPropertyChanged()函数。

如果用户没有为某个类创建类似的函数，fody.PropertyChanged会为该类动态生成 这个函数。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
public void FindMethodsForNodes()
{
  foreach (var notifyNode in NotifyNodes)
  {
    var eventInvoker = RecursiveFindEventInvoker(notifyNode.TypeDefinition);
    if (eventInvoker == null)
    {
      eventInvoker = AddOnPropertyChangedMethod(notifyNode.TypeDefinition);
      ...
    }
    notifyNode.EventInvoker = eventInvoker;
    foreach (var childNode in notifyNodes)
    {
      ProcessChildNode(childNode, eventInvoker);
    }
  }
}

该函数遍历节点链中的每个节点，找到该节点对应的类中的类似NotifyPropertyChanged() 的方法，记录对该方法的引用到节点的EventInvoker属性中。

如果NotifyPropertyChanged()方法不存在，就为该节点对应的类动态生成一个。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
public EventInvokerMethod RecursiveFindEventInvoker(TypeDefinition typeDefinition)
{
  var typeDefinitions = new Stack<TypeDefinition>();
  MethodDefinition methodDefinition;
  var currentTypeDefinition = typeDefinition;
  do
  {
    typeDefinitions.Push(currentTypeDefinition);
    if (FindEventInvokerMethodDefinition(currentTypeDefinition, out methodDefinition))
    {
      break;
    }
    var baseType = currentTypeDefinition.BaseType;
    if (baseType == null || baseType.FullName == "System.Object")
    {
      return null;
    }
    currentTypeDefinition = Resolve(baseType);
  } while (true);
  return new EventInvokerMethod
  {
    MethodReference = GetMethodReference(typeDefinitions, methodDefinition),
    IsVisibleFromChildren = IsVisibleFromChildren(methodDefinition),
    InvokerType = ClassifyInvokerMethod(methodDefinition),
  };
}

这个方法试图找到传入节点对应的类的NotifyPropertyChanged()方法，找到了 会返回一个EventInvokerMethod对象。如果找不到，就尝试在该类的父类中寻找。 如果还找不到，就尝试其父类的父类，依次类推，直到当前类为System.Object 为止，然后返回null。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
bool FindEventInvokerMethodDefinition(TypeDefinition type, out MethodDefinition methodDefinition)
{
  methodDefinition = type.Methods
    .Where(x => (x.IsFamily || x.IsFamilyAndAssembly || x.IsPublic || x.IsFamilyOrAssembly) && EventInvokerNames.Contains(x.Name))
    .OrderByDescending(definition => definition.Parameters.Count)
    .FirstOrDefault(x => IsBeforeAfterGenericMethod(x) || IsBeforeAfterMethod(x) || IsSingleStringMethod(x) || IsPropertyChangedArgMethod(x) || IsSenderPropertyChangedArgMethod(x));
  if (methodDefinition == null)
  {
    methodDefinition = type.Methods
      .Where(x => EventInvokerNames.Contains(x.Name))
      .OrderByDescending(definition => definition.Parameters.Count)
      .FirstOrDefault(x => IsBeforeAfterGenericMethod(x) || IsBeforeAfterMethod(x) || IsSingleStringMethod(x) || IsPropertyChangedArgMethod(x) || IsSenderPropertyChangedArgMethod(x));
  }
  return methodDefinition != null;
}

public static bool IsPropertyChangedArgMethod(MethodDefinition method)
{
  var parameters = method.Parameters;
  return parameters.Count == 1

public static bool IsSingleStringMethod(MethodDefinition method)
{
  var parameters = method.Parameters;
  return parameters.Count == 1
    && parameters[0].ParameterType.FullName == "System.String";
}

public static bool IsBeforeAfterMethod(MethodDefinition method)
{
  var parameters = method.Parameters;
  return parameters.Count == 3
    && parameters[0].ParameterType.FullName == "System.String"
    && parameters[1].ParemeterType.FullName == "System.Object"
    && parameters[2].ParameterType.FullName == "System.Object";
}

public static bool IsBeforeAfterGenericMethod(MethodDefinition method)
{
  var parameters = method.parameters;
  return parameters.Count == 3
    && method.HasGenericParameters
    && method.GenericParameters.Count == 1
    && parameters[0].ParameterType.FullName == "System.String"
    && parameters[1].ParameterType.FullName == method.GenericParameters[0].FullName
    && Parameters[2].ParameterType.FullName == method.GenericParameters[0].FullName;
}

// PropertyChanged\PropertyChanged.Fody\EventInvokerNameResolver.cs
public List<string> EventInvokerNames = new List<string>
{
  "OnPropertyChanged",
  "SetProperty",
  "NotifyOfPropertyChange",
  "RasiePropertyChanged",
  "NotifyPropertyChanged",
  "NotifyChanged",
  "raisePropertyChanged"
};

从上面的代码中可以看出，虽然在文章开头的例子中使用的是NotifyPropertyChanged(string propertyName)， 但该方法的名字可以是EventInvokerNames数组中的任意一个，而且该方法的 参数也可以是不同数量和不同类型的。

例如，假设该方法是名字是NotifyPropertyChanged，它的参数可以是以下任意 一种，

  • NotifyPropertyChanged(string propertyName)
  • NotifyPropertyChanged(string propertyName, object beforeValue, object afterValue)
  • NotifyPropertyChanged(string propertyName, T beforeValue, T afterValue)
  • NotifyPropertyChanged(PropertyChangedEventArgs arg)
  • NotifyPropertyChanged(obejct sender, PropertyChangedEventArgs arg)

注意，在寻找这个方法时，是有顺序的，如果一个类同时拥有不同上述方法中 的一个，以第一个找到的为准。

当找到一个符合条件的NotifyPropertyChanged()方法，RecursiveFindEventInvoker()方法 会返回一个EventInvokerMethod对象。

PropertyChanged\PropertyChanged.Fody\EventInvokerMethod.cs
public class EventInvokerMethod
{
  public MethodReference MethodReference;
  public InvokerTypes InvokerType;
  public bool IsVisibleFromChildren;
}

代码先去寻找方法的引用。

// PropertyChanged\PropertyChanged.Fody\MethodGenerifier.cs
public MethodReference GetMethodReference(Stack<TypeDefinition> typeDefinitions, MethodDefinition methodDefinition)
{
  var methodReference = ModuldeDefinition.ImportReference(methodDefinition).GetGeneric();
  typeDefinitions.Pop();
  while (typeDefinitions.Count > 0)
  {
    var definition = typeDefinitions.Pop();
    methodReference = MakeGeneric(definition.BaseType, methodReference);
  }
  return methodReference;
}

public static MethodReference MakeGeneric(TypeReference declaringType, MethodReference self)
{
  var reference = new MethodReference(self.Name, self.ReturnType)
  {
    DeclaringType = declaringType,
    HasThis = self.HasThis,
    ExplicitlyThis = self.ExplicitlyThis,
    CallingConvention = self.CallingConvention,
  };
  foreach (var parameter in self.Parameters)
  {
    reference.Parameters.Add(new ParameterDefinition(parameter.ParameterType));
  }
  return reference;
}

这段代码的逻辑是，如果typeDefinitions栈中有两个以上的元素，就说明 NotifyPropertyChanged()方法声明在节点对应的类中的父类或父类的父类中， 所以得到的方法的引用是对父类或父类的父类的方法的引用。

这里不明白为什么在GetMethodReference()方法中使用循环。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
static bool IsVisibleFromChildren(MethodDefinition methodDefinition)
{
  return methodDefinition.IsFamilyOrAssembly || methodDefinition.IsFamily || methodDefinition.IsFamilyAndAssembly || methodDefinition.IsPublic;
}

这里判断NotifyPropertyChanged()方法是否在节点对应的类的子类中是可见的。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
public static InvokerTypes ClassifyInvokerMethod(MethodDefinition method)
{
  if (IsSenderPropertyChangedArgMethod(method))
  {
    return InvokerTypes.SenderPropertyChagnedArg;
  }
  if (IsPropertyChangedArgMethod(method))
  {
    return InvokerTypes.PropertyChangedArg;
  }
  if (IsBeforeAfterMethod(method))
  {
    return InvokerTypes.BeforeAfter;
  }
  if (IsBeforeAfterGenericMethod(method))
  {
    return InvokerTypes.BeforeAfterGeneric;
  }
  return InvokerTypes.String;
}

这里判断找到的NotifyPropertyChanged()方法的签名的类型，为了以后使用时方便。

如果节点对应的类中或该类的父类或父类的父类中不存在类似NotifyPropertyChanged() 方法，fody.PropertyChanged会为该类注入一个简单的NotifyPropertyChanged()方法。 注入代码的细节在这里省略。

我们回到FindMethodForNodes()方法中，当找到了当前节点对应的类的NotifyPropertyChanged() 方法后，再继续处理该类的每个子类。

// PropertyChanged\PropertyChanged.Fody\MethodFinder.cs
void ProcessChildNode(TypeNode node, EventInvokerMethod eventInvoker)
{
  var childEventInvoker = FindEventInvokerMethod(node.TypeDefinition);
  if (childEventInvoker == null)
  {
    if (node.TypeDefinition.BaseType.IsGenericInstance)
    {
      var methodReference = MakeGeneric(node.TypeDefinition.BaseType, eventInvoker.MethodReference);
      eventInvoker = new EventInvokerMethod
      {
        InvokerType = eventInvoker.InvokerType,
        MethodReference = methodReference,
        IsVisibleFromChildren = eventInvoker.IsVisibleFromChilren
      };
    }
  }
  else
  {
    eventInvoker = childEventInvoker;
  }
  ...
  node.EventInvoker = eventInvoker;
  foreach (var childNode in node.Nodes)
  {
    ProcessChildNode(childNode, eventInvoker);
  }
}

EventInvokerMethod FindEventInvokerMethod(TypeDefinition type)
{
  if (!FindEventInvokerMethodDefinition(type, out var methodDefinition))
  {
    return null;
  }
  var methodDefinition = ModuleDefinition.ImportReference(methodDefinition);;
  return new EventInvokerMethod
  {
    MethodReference = methodReference.GetGeneric(),
    IsVisibleFromChildren = IsVisibleFromChildren(methodDefinition),
    InvokerType = ClassifyInvokerMethod(methodDefinition)
  };
}

对于某个类的子类来说，因为存在父类与子类的关系，所以如果子类没有类似 NotifyPropertyChanged()方法，它可以调用父类的NotifyPropertyChanged()方法， 所以fody.PropertyChanged不需要为它动态生成一个。

这里先检索子类是否存在NotifyPropertyChanged()方法，如果有，就记录该方法 的引用在子节点中。如果没有，就记录父节点的方法在子节点中。

当为当前节点找到适合的方法引用后，ProcessChildNode()递归继续处理该节点 的子节点。
